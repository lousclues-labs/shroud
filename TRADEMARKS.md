The following uses are permitted without written authorization:

- **Descriptive reference** — Referring to the Shroud project by name in articles, blog posts, documentation, or conversations (e.g., "compatible with Shroud," "tested against Shroud").
- **Attribution** — Stating that your project is "based on Shroud" or "derived from Shroud" when this is factually accurate.
- **Linking** — Linking to the official repository (https://github.com/loujr/shroud) or website (https://vpnshroud.org).
- **Academic and research use** — Referencing Shroud in academic papers, benchmarks, or security research.
- **Package naming** — Using the name "shroud" in system package repositories (e.g., AUR, Debian) when distributing the unmodified or minimally patched official release, provided it links back to the official repository.

When using the names in any of the above contexts, you should use them as proper nouns (capitalize "Shroud") and should not alter the spelling or combine them with other words in a way that creates a new mark.

## Forks and Derivative Works

If you fork or create a derivative work from Shroud's source code (as permitted by the GPL), you must:

1. **Choose a different name** for your project that does not include "Shroud" or "lousclues."
2. **Remove or replace** any Shroud or lousclues logos, icons, or branding from your fork.
3. **Clearly indicate** in your README and documentation that your project is a fork or derivative and is not the official Shroud project.
4. **Include attribution** — State that your project is derived from Shroud and link to the official repository.

**Acceptable examples:**
- ✅ "MyVPNManager — a fork of Shroud"
- ✅ "VeilNet, based on Shroud (https://github.com/loujr/shroud)"

**Unacceptable examples:**
- ❌ "Shroud Enhanced"
- ❌ "Shroud Community Edition"
- ❌ "Shroud for Windows"
- ❌ "Shroud 2.0"
